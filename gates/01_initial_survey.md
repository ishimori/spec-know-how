# Gate 1: 初回調査

**確認タイミング**: Step 1（対象を把握する）完了後

次のステップに進む前に、以下の項目を確認してください。

---

## コードベースの把握

- [ ] ディレクトリ構造を確認し、主要フォルダの役割がわかる
- [ ] 技術スタックを確認した（言語・フレームワーク・バージョン）
- [ ] LOC（行数）およびファイル数を把握した
- [ ] 主要なエントリポイント（main.py、index.ts 等）を特定した

## 既存ドキュメントの把握

- [ ] README や既存ドキュメントの有無を確認した
- [ ] ドキュメントとコードの間に明らかな食い違いがないか確認した（ある場合はメモした）
- [ ] コードコメントや docstring の量・質を確認した

## 体制・環境の把握

- [ ] CI/CD の有無と構成を確認した（ある場合）
- [ ] 本番・ステージング・開発環境の構成を確認した（ある場合）
- [ ] コードの所有者・担当者を確認した（ある場合）

## DA 批判レビュー（推奨）

- [ ] このステップの成果物に対して DA 批判レビューを実施した（少なくとも1件の問題を発見・対応）

> DA 批判レビューの詳細は [manuals/01_仕様書作成マニュアル.md](../manuals/01_仕様書作成マニュアル.md) を参照。

---

## メモ欄

確認した内容や気づき、追加調査が必要な点を記録してください。

```
（例）
- フロントエンドとバックエンドが同一リポジトリに混在している
- README は2年前から更新されていない模様。フレームワークのバージョン情報が古い
- テストコードはほぼなし
```
