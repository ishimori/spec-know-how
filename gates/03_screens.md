# Gate 3: 画面・UI

**確認タイミング**: Step 3（画面・UIを把握する）完了後

次のステップに進む前に、以下の項目を確認してください。

---

## 画面一覧の整備

- [ ] 画面一覧を作成した（画面名・URL/パス・主な機能）
- [ ] モーダル・ダイアログなど補助的な画面も含めて把握した
- [ ] 画面に紐づく API エンドポイントを把握した

## 画面遷移と操作フロー

- [ ] 主要な操作フロー（Happy Path）を把握した
  - 例: 申請→承認→通知、入力→確認→完了
- [ ] 画面遷移を図または文章で整理した

## 権限・ロール制御

- [ ] 権限・ロールによる表示制御の有無を確認した
- [ ] 同じ画面でも権限によって操作できる内容が変わる場合、その条件を把握した

## ビジネスロジックの在処

- [ ] submit/save 関数に業務ロジック・バリデーションが含まれていないか確認した
  - UI 層でもビジネスルールを持つことがある

> **なぜ重要か**: submit 関数に埋まったバリデーション条件が丸ごと見落とされたケースがある。「画面を作る」と「ルールを把握する」を明示的に分けて確認する。
