# SP-003-context: 画面仕様書ファイル構成設計

## このチケットの位置づけ

**全体目標**: このSPシリーズは [manuals/01_仕様書作成マニュアル.md](../../manuals/01_仕様書作成マニュアル.md) の Step 3（画面・UI を把握する）完了を目指す — 画面構成・遷移・操作フローを把握して後工程の仕様書作成の根拠とする

**マニュアル対応**: 特定の Step に非対応（全工程を横断）— SP-003-01〜03（および動的追加チケット）の成果物を集約し、永続的な画面仕様書ファイルの構成を設計する

**目的**: SP-003-01〜03（および動的追加チケットを含む全子チケット）の調査結果を統合し、後続の SP が参照する画面仕様書ファイルの構成・ファイル名・記述内容を設計する

**最終的な使われ方**: ここで設計したファイル構成に従って画面仕様書を作成する（実際のファイル作成はこのチケット内で行う）。作成したファイルは `CLAUDE.md` または `document_map.md` に追記し、SP-004 以降から参照可能にする

**実行 Phase の判断**:

| Phase | 実行条件 |
|-------|---------|
| Phase 1 | 常に実行（SP-003-01〜03 および動的追加チケット完了後） |
