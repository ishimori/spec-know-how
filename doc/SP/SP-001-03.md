## このチケットの位置づけ

**全体目標**: このSPシリーズは [manuals/01_仕様書作成マニュアル.md](../../manuals/01_仕様書作成マニュアル.md) の Step 0〜4 完了を目指す — レガシーシステムから [VERIFIED] 仕様書を抽出し、生成AIによるリエンジニアリングを可能にする。

**マニュアル対応**: [manuals/01_仕様書作成マニュアル.md](../../manuals/01_仕様書作成マニュアル.md) — **Step 1「ドキュメント棚卸し」本体**（3.1 手順）

**目的**: 既存ドキュメントの全量把握・信頼度ラベル付与と、環境設定・体制の把握。
ここで付与した信頼度ラベルが SP-002 以降の調査優先度を決める
（UNVERIFIED 多数 → 機械処理強化、VERIFIED 多数 → 文書ベース調査が有効）。

**最終的な使われ方**: ここで付与した信頼度ラベルと環境情報は SP-001-context の設計根拠となり、
**SP-002 以降の AI セッションが「どのドキュメントをどの程度信頼すべきか」を判断する基準**となる。

**実行 Phase の判断**:

| Phase | 実行条件 |
|-------|---------|
| Phase 1: 既存ドキュメントの棚卸し | SP-000 グループB「ドキュメントあり / 一部あり」の場合のみ。**なし** の場合はスキップ |
| Phase 2: 体制・環境の把握 | **常に実行** |

## Phase 1: 既存ドキュメントの棚卸し

> **実行条件**: SP-000 グループB「ドキュメント有無」が **あり / 一部あり** の場合のみ。**なし** の場合はこの Phase をスキップ。

> チェックを入れるだけでなく、調査記録テーブルに内容を記録すること。

- [ ] README や既存ドキュメントを確認し、信頼度ラベルを付与した
- [ ] ドキュメントとコードの間に明らかな食い違いがないか確認した（ある場合はメモした）
- [ ] コードコメントや docstring の量・質を確認した

### 調査記録

|-------------------|------|------------|----------------------|
| - | - | VERIFIED / DRAFT / UNVERIFIED | - |

## Phase 2: 体制・環境の把握

> チェックを入れるだけでなく、調査記録テーブルに内容を記録すること。

- [ ] CI/CD の有無と構成を確認した（ある場合）
- [ ] 本番・ステージング・開発環境の構成を確認した（ある場合）
- [ ] 環境設定ファイル（.env.example / config 等）の有無を確認した
- [ ] ビルド/デプロイ設定（Dockerfile / docker-compose 等）の有無を確認した
- [ ] コードの所有者・担当者を確認した（ある場合）

| 環境構成（本番 / ステージ / 開発） | - | - |
| 環境設定ファイルの有無・場所 | - | - |
| ビルド/デプロイ設定の有無・場所 | - | - |
| コードの所有者・担当者 | - | - |
