## 目的

Claude Code を知らない開発者が README.md を読んで「このリポジトリが何をするものか・自分がどう使うか」を理解できるようにする。
新規ドキュメントは作らず、README.md の改修のみで解決する。

## 背景・課題

現在の README.md は既存ユーザー（DD スキルや Claude Code を知っている人）向けには十分だが、初見の開発者には以下の障壁がある：

1. **「Claude Code とは何か」が説明されていない** — ツール名が突然登場する
2. **「DD とは何か」が唐突** — GUIDE.md に入った瞬間から DD 前提の話になる
3. **このリポジトリがプロジェクト全体のどこに位置するかが見えない** — レガシー移行のどのフェーズで使うのかが大局的に掴めない

## 検討内容

### 選択肢の比較

| 案 | 内容 | 判断 |
|----|------|------|
| A: 別途「概要ドキュメント」を新設 | `docs/overview.md` 等を追加 | ❌ ドキュメントが分散する。README が入口であるべき |
| B: README.md に図と説明を追加 | Mermaid 図 + 用語説明を README に組み込む | ✅ 採用 |

### 追加すべき内容

- **大局観を示す Mermaid 図**: レガシーシステム → 仕様抽出（spec-know-how）→ 実装 → 新システム、という流れの中での位置付け
- **「Claude Code とは」1行説明**: 注釈レベルでよい
- **「DD とは」1行説明**: README 内で初出時に補足する

### 追加しないもの

- Claude Code のセットアップ手順（外部リンクで十分）
- 詳細な使い方チュートリアル（GUIDE.md の役割）

## 決定事項

README.md のみを改修する。追加内容は以下の3点に絞る：

1. 冒頭に「全体の位置付け」Mermaid 図を追加
2. 「Claude Code」初出箇所に1行注釈を追加
3. 「DD」初出箇所に1行注釈を追加
